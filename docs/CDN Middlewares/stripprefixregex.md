# Strip Prefix Regex

حذف پیشوندها از مسیر قبل از ارسال درخواست (با استفاده از Regex). این میان افزار پیشوند مسیر تطبیق را از بین می برد و آن را در هدر `X-Forwarded-Prefix` ذخیره می کند. اگر backend شما به مسیر اصلی (/) گوش می دهد اما باید روی یک پیشوند خاص در معرض دید قرار گیرد، از یک میان افزار `stripPrefixRegex` استفاده کنید. .

## Regex

گزینه regex، یک regex برای مطابقت با پیشوند مسیر از URL درخواست است.

به عنوان مثال، products/ همچنین با products/shoes/ و products/shirts/ مطابقت دارد.

اگر backend شما assets (مانند تصاویر یا فایل‌های جاوا اسکریپت) را ارائه می‌کند، می‌تواند از هدر `X-Forwarded-Prefix` برای ساخت صحیح URL‌های نسبی استفاده کند. با استفاده از مثال قبلی، backend باید products/shoes/image.png/ را برگرداند (و نه images.png/، که احتمالا دژ نمی تواند با آن backend مرتبط شود).
